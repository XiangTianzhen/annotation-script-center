# Aishell Tech 闽南语助手运行时

## 当前范围

- 仅在 `https://mark.aishelltech.com/mytask/mark?...` 生效。
- 前端显示名当前统一使用 `希尔贝壳`；仅改用户可见文案，不改内部平台 ID、文件夹名、接口路径或后端注册名。
- 当前默认配置已对齐 DataBaker 一检质检：`two_stage + mandarin_to_dialect + qwen3.5-omni-flash + qwen3.5-plus`。
- options 页当前已与 DataBaker 共用固定顺序的右侧 `AI 设置` 模块；`AI 连续填入并发数量` 已移到该区域，默认 `5`，Omni 范围 `1~25`，Fun-ASR 范围 `1~50`。
- 实测验收建议先打开 `https://mark.aishelltech.com/mytask/detail/:taskId`，再点击分包“查看”进入标注页；直接输入 `/mytask/mark?...` 在平台侧可能出现卡住。
- 当前前端已按“嵌入式卡片与原生栏注入”集成：
  - 面板以相对布局自适应嵌入在原表单正下方。
  - 面板卡片本体只保留推荐展示、状态和详细信息折叠，不再堆叠主动操作区。
  - `AI识别` 按钮直接注入到页面原生“保存”按钮右侧。
  - `全部AI批量识别`、`未完成的AI批量识别` 与 `停止批量` 按钮直接注入到页面原生工具按钮区域，并跟随页面重渲染自动恢复。
  - 当 AI 推荐文本与页面参考文本一致时，面板会显示 `无需修改` 提示，但底层仍保留真实推荐文本用于后续填入。
  - “当前识别结果”区会额外显示 AI 诊断信息：识别策略、模型选择、AI耗时、前端并发、输入/输出/总 token、FunASR provider、后端模式、后端地址、是否发生自动回退、requestId、debugId。
  - 诊断展示当前统一优先读取后端 `meta`：额外包含排队等待、缓存命中和当前阶段，不再依赖旧的零散字段拼装。
  - 如果当前后端模式是“本机（127.0.0.1:3333）”且本机接口不可达，前端会自动回退一次 `script.xiangtianzhen.store` 服务器接口；只影响本次请求，不会改写用户设置。
  - 如果 AI 请求在浏览器层直接失败（例如本机后端没启动、网络层失败、扩展上下文失效），状态区会直接展开原始诊断 JSON，至少包含：当前后端模式、请求 endpoint、回退 endpoint、浏览器原始异常名/异常消息、在线状态。
  - 当前错误展示已区分两类：
    - 系统/网络类错误：原始 JSON 会收口为精简中文诊断，只保留错误原因、接口地址、health 状态和排查建议。
    - AI/上游模型类错误：继续完整保留后端原始返回，并额外给出中文“错误解读/可能原因”。
  - 当前共享错误解读规则已覆盖：
    - `429 + limit_burst_rate`：按“上游模型限流 / 请求增长过快”解释。
    - `400 + Arrearage`：按“账号欠费或余额不足”解释。
    - 其他 `400`：不直接推断为余额问题，优先按请求参数或上游调用条件错误处理。
  - 当 `POST /recommend` 直接 `Failed to fetch` 时，前端还会立刻再探测一次 `/recommend/health`；如果 health 正常，会明确提示“入口活着，但真实推荐请求链路在网络层被中断”，方便优先排查 Nginx / PM2 / Node 进程日志。
  - 该形态作为 Aishell 前端的默认长期样式基线，后续迭代优先在这套嵌入式结构上继续细化。
- 已移除 Aishell 的高风险注入链：
  - 移除 `page-world/network-observer.js` 注入。
  - 不再依赖页面抓包缓存。
- 当前取数方式改为：
  - 内容脚本直接从页面 `localStorage/sessionStorage` 里扫描 JWT。
  - 直接请求 `markapi.aishelltech.com` 的 `task/detail` 与 `packageItemList`。
  - `detail -> mark` 只靠同一个 content script 的 URL 轮询识别路由切换，不再额外二次注入。
  - AI 请求会额外从头像下拉 `.avatar-dropdown .user-name .hidden-xs-only` 读取平台账号，并把 `ASmnbz001【标注人员】` 这类显示文本归一成纯账号 `ASmnbz001`。
- 当前条识别支持：
  - 读取当前选中条。
  - 请求后端 AI recommend。
  - 在悬浮窗里展示听音文本与推荐文本。
  - 支持复制听音文本、复制推荐文本、填入并保存当前条、忽略本次结果。
  - 支持在 options 中给以上动作配置快捷键。
  - 若页面是旧 content script 上下文（例如重新加载 unpacked extension 后未刷新业务页），单条识别会直接提示 `扩展上下文已失效，请刷新当前业务页面后重试。`，不再误报成普通“后端连接中断”。
- 批量识别支持：
  - 每次启动后都从当前分包第 1 条扫到最后 1 条。
  - 页面当前提供两种批量入口：
    - `全部AI批量识别`：整包扫描，不跳过 `dataStatus === 2` 的已完成条目。
    - `未完成的AI批量识别`：沿用原行为，只处理未完成条目。
  - 先通过 `packageItemList` 直接读取当前分包可处理条目，不等待页面切到当前条后再发 AI 请求。
  - `未完成的AI批量识别` 会跳过 `dataStatus === 2` 的已完成条目；默认 `0` 视为待处理。
  - AI 请求按配置并发发起，但每次请求固定错峰 `50ms`，整体发送速率不超过 `20 req/s`。
  - 批量窗口采用“固定并发池 + 保存完成后补位”：
    - 先按并发数发起首批识别请求。
    - 哪条先返回，就先进入保存队列。
    - 某条完成填入并保存后，才从剩余待处理条目中补发下一条识别请求。
  - 保存阶段会按条目编号 + 文件名后缀重新在左侧列表里定位目标条，再切换右侧表单。
  - 右侧当前条校验会优先从 `.fileName-line` 整行提取 `编号: 文件名.wav`，兼容平台把编号单独放在第一个 `span`、文件名与工具按钮放在同一行的结构，避免误判“右侧表单未切换”。
  - 填入后统一触发页面真实“保存”按钮，不再由内容脚本自行直发 `SaveShortMark`。
  - 保存成功判定优先看页面 `保存成功!` 提示；若提示未及时出现，再回退检查条目完成态与 `getShortMark / packageItemList` 是否已更新。
  - 支持用户手动停止当前批次。
  - 保留失败清单，便于回看失败条目。
  - 每条失败项旁边都会显示 `查看详情` 与 `查看原始JSON`：
    - `查看详情` 会展开失败阶段、错误摘要、AI耗时、模型、前端并发、token、排队等待、缓存命中、后端模式、后端地址、是否自动回退、requestId、debugId 等诊断信息。
    - `查看原始JSON` 会展开该条完整失败上下文；AI 请求失败时优先显示后端原始返回，若是浏览器层网络失败则显示当前 endpoint / 回退 endpoint / 原始异常 / online 状态，切条/保存失败时显示对应链路上下文与 AI debug 信息。

## 当前后端契约

- 成功响应固定为 `success + data + meta`。
- 失败响应固定为 `success=false + error + meta`。
- 前端当前继续向上兼容旧展示结构，但诊断信息统一从 `meta` 派生：
  - `meta.models`
  - `meta.timing`
  - `meta.usage`
  - `meta.queue`
  - `meta.cache`
  - `meta.cancelled`

## 识别策略

- `普通话对照默认`：
  - 听音模型先把闽南语音频转成普通话文本。
  - 比较/转换模型再结合页面预测闽南语文本和字词对照表，输出最终闽南语推荐文本。
  - 默认 Prompt 已限制 `recommendedText` 必须使用简体中文，不允许输出繁体字；前端不再做二次转换。
- `直接听写闽南语`：
  - 听音模型直接输出闽南语文本。
  - 比较模型再结合页面预测文本做最终闽南语推荐。
  - 默认 Prompt 同样要求 `heardText / recommendedText` 使用简体中文，不允许输出繁体字。

## 边界

- 当前最小重构版：只有用户明确点击 `填入并保存当前条`、`全部AI批量识别` 或 `未完成的AI批量识别` 后，才允许点击页面真实“保存”。
- 不自动提交任务。
- 不跨分包处理。
- 不触发 `.check-area`。
- 不再使用主世界抓包或动态脚本补注入。
- AI 调用使用人仍来自 options 首页全局设置；平台账号只从当前页面头像区自动提取，不在前端本地额外持久化。
