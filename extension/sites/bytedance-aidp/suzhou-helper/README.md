# ByteDance AIDP 苏州话脚本运行时

## 命中范围

- 站点：`https://aidp.bytedance.com/*`
- 目标页：`/management/task-v2/{taskId}/mark-v3/{index}`

## 当前能力

- 读取脚本中心里的基础设置：
  - `隐藏平台AI功能`
  - `画段后自动应用建议`
  - `前后静音时长`
  - `静音阈值`
  - `连续相接自动合并`
  - `默认播放倍数`
  - `固定缩放倍数`
  - `启用普通话听写识别`
  - `AI 请求超时`
  - `听音 / 普通话听写收口` 两阶段模型、Prompt 与生成参数
  - `快捷键`
- 仅在 `mark-v3` 详情页生效。
- 当脚本中心勾选 `隐藏平台AI功能` 时，自动隐藏两类平台原生 AI 板块：
  - 左侧 `AI 洞察` 卡片
  - 右下角平台 AI 浮动入口
- 页面首次加载和波形区重渲染后，会继续处理播放速度控件和缩放输入框：
  - 播放速度只在进入当前详情页或切到新题时按一次配置值走平台原生提交
  - 若点选目标倍速后平台展示值仍未变化，会补一次原生确认动作
  - 缩放只在进入当前详情页时按一次配置值；优先读取 `aria-valuenow`，再按平台原生缩小 / 放大按钮收敛到整数 `1~10`
- 当前会在分段表格里补挂行内识别入口：
  - 表头新增一列 `识别音频`
  - 每行在现有 `暂停` 右侧新增一个 `识别音频` 按钮
  - 当设置页开启 `识别完成后立即填入` 时，点击后会直接填入该行输入框
  - 当设置页关闭 `识别完成后立即填入` 时，首次点击只识别并缓存结果，按钮切换为 `填入`，再次点击才会写入该行输入框
  - 成功后底部 `AI信息` 左侧会展示本次 `最终普通话听写稿`、`听音阶段结果`、阶段级 token / 预估人民币，右侧展示 `AI返回内容`，并提供 `复制内容`
  - 当前会在波形区下方挂载 `苏州话脚本` 面板，固定为 4 个区块：
  - 第一排左侧：`分段建议`
    - 调用统一后端生成当前音频分段建议
    - 根据设置页里的 `画段后自动应用建议` 决定显示 `生成分段并应用` 或 `生成分段建议 / 应用分段建议`
    - 分段建议结果区默认折叠，展开后按 `建议段 N：开始 X 秒，结束 Y 秒` 卡片展示
    - 标题帮助说明统一使用纯 `?` 提示入口
    - 通过页面内捕获的 `SubmitTempItemAnswer` 暂存契约写回
  - 第一排右侧：`批量识别`
    - 只处理当前题当前页 `regions`
    - 默认全选，支持段号点选和拖选
    - 默认并发 `5`
    - 主按钮按状态在 `批量识别并填入 / 批量识别 / 停止批量 / 填入` 之间切换
    - 当设置页开启 `识别完成后立即填入` 时，整批识别结束后会直接写回成功且非空的段
    - 当设置页关闭 `识别完成后立即填入` 时，整批识别结束后先缓存结果，点击 `填入` 后再统一调用一次 `SubmitTempItemAnswer`
  - 第二排整宽：`当前音频信息`
  - 第三排整宽：`AI信息`
    - 展示行内识别或批量识别最近一次结果的两阶段模型、usage、cost 和 debug
    - 若最近一次结果来自批量识别，顶部会按段号顺序显示切换按钮，点击后切换到对应段的 AI 信息
- 当前会在题目头部信息条右侧的 `.operation-group-btn-GcvnvK` 挂两个辅助按钮：
  - `清空画段`
  - `填充语言种类`
- 当前支持 7 个详情页快捷键动作：
  - `播放/暂停切换`
  - `区间播放`
  - `回到首帧`
  - `删除当前选区`
  - `清空画段`
  - `生成分段建议`
  - `应用分段建议`

## 写回契约

- 单段识别会直接填入当前分段表的输入框，不主动发平台暂存请求
- 批量识别与分段建议应用固定写入 `TempAnswer.Content.data.regions[*].txt`
- 最终结果字段固定为 `finalMandarinText`
- 不新增第二个结果字段
- 不改 `regions[*].ms`
- 不改提交 / 下一题链路
- 空结果处理：
  - 目标段原本已有非空 `txt` 时，空结果不覆盖，直接跳过
  - 目标段原本为空时，空结果视为 no-op

## 文本口径

- 最终输出是“普通话听写稿”，不是苏州话原文稿，也不是润色稿
- 普通话不要截取，听到多少写多少
- 以听音为主，不做语义扩写
- 标点只允许使用 `，。？！`，句末只允许使用 `。？！`
- 不知名人名、地名、公司名或其他无法精准锁定的事物，使用 `##名称##` 包起来
- 抖音音效不截取；主说话人如果在唱歌，也不截取这部分内容
- 语气词等按听到的普通话写法保留
- 明显口吃式连续重复最多保留 `3` 次
- 正常有语义的重复内容按听到的保留
- 不使用阿拉伯数字，统一改写为汉字数字
- 纯静音或完全听不清时返回空字符串

## 分段留白

- `前后静音时长` 的运行时补偿始终限制在 `0 ~ 500ms`
- 截取区间开头和结尾的无人声留白都不能超过 `500ms`

## 运行时边界

- 当前统一后端接口：
  - `/api/bytedance-aidp/suzhou-helper/segment/preview`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend/health`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend/defaults`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend`
- 当前批量识别只处理当前题当前页，不跨页、不自动切题
- 当前批量识别与分段建议写回只作用于 `SubmitTempItemAnswer` 暂存答案
- 当前不会直接点击平台 `提交 / 下一题 / 重置`
- 当前不会新增平台提交 API
- 当前只改 `txt`，不改 `ms`
- 当前若发现当前题或当前快照已变化，批量写回会 fail closed 取消

## 文件

- `page-world/network-observer.js`
  - 捕获 `Receive` 当前条读取快照
  - 捕获 `SubmitTempItemAnswer` 暂存请求 URL、CSRF 头和基础请求体
- `data-api.js`
  - 解析详情页当前上下文
  - 按段号直填当前输入框
  - 通过平台暂存接口写回批量 `txt`
  - 应用分段建议
  - 清空画段
  - 填充语言种类
- `ai-recommendation.js`
  - 裁当前段音频
  - 调统一后端苏州话 AI 推荐接口
  - 复用共享 AI 使用人元数据
- `segmentation-controller.js`
  - 调统一后端生成分段建议
- `ui-panel.js`
  - 渲染分段建议 / 批量识别 / 当前音频信息 / AI 信息区块
- `shortcuts.js`
  - 绑定 AIDP 详情页快捷键
- `content.js`
  - `mark-v3` 路由识别
  - 平台 AI 板块显隐
  - 波形区播放速度 / 固定缩放回填
  - 分段表格行内识别按钮挂载
  - 面板协调、识别、批量识别与按钮接线

## 真实浏览器验证

1. 在仓库根目录启动统一后端：`node platform-resources/backend/server.js`
2. 在真实 Edge / Chrome 重新加载 unpacked extension，并刷新 `mark-v3` 详情页。
3. 进入脚本中心 `ByteDance AIDP -> 苏州话脚本`，确认基础设置和 AI 设置都能正确回显。
4. 回到详情页，确认左侧 `AI 洞察` 卡片和右下角平台 AI 浮动入口按开关显隐。
5. 确认 `苏州话脚本` 第一排左侧是 `分段建议`、右侧是 `批量识别`，下方依次是 `当前音频信息` 和 `AI信息`，且不再出现旧版单段识别和手动写回入口。
6. 在分段表格里确认只有存在真实分段时才会新增 `识别音频` 列，并且每行 `暂停` 右侧都有行内按钮；空表格时不出现该列和按钮。
7. 若设置页开启 `识别完成后立即填入`，点击任意一行 `识别音频` 后应直接更新该行输入框内容，不改 `ms`；若关闭该开关，则首次点击后按钮切成 `填入`，再次点击才写入输入框。
8. 确认底部 `AI信息` 左侧展示 `最终普通话听写稿`、`听音阶段结果` 和 usage/cost，右侧显示 `AI返回内容` 与 `复制内容`，且不再显示 `raw.listen / raw.refine`。
9. 验证纯静音或完全听不清时：
  - 原本已有 `txt` 的段不会被空结果覆盖
  - 原本为空的段保持不写
10. 验证 `批量识别并填入`：
  - 全选可用
  - 段号点选和拖选可用
  - `停止批量` 可用
  - 自动填入开启时，主按钮应在 `批量识别并填入 / 停止批量` 之间切换
  - 自动填入关闭时，主按钮应在 `批量识别 / 停止批量 / 填入` 之间切换
  - 批量结果会在 `AI信息` 顶部显示按段号排序的切换按钮
  - 最终只发一次批量写回
11. 关闭设置页 `画段后自动应用建议` 后，确认运行时显示 `生成分段建议` 和 `应用分段建议` 两个按钮；开启后只显示 `生成分段并应用`。
12. 点击题目头部右侧的 `清空画段` 和 `填充语言种类`，确认旧能力不回归，且波形区原有播放 / 缩放图标不再承载这两个辅助按钮。
