# ByteDance AIDP 苏州话脚本运行时

## 命中范围

- 站点：`https://aidp.bytedance.com/*`
- 目标页：`/management/task-v2/{taskId}/mark-v3/{index}`

## 当前能力

- 读取脚本中心里的基础设置：
  - `隐藏平台AI功能`：默认勾选并隐藏平台 AI 板块，取消勾选后才显示
  - `前后静音时长`：默认 `0.5s`，可调范围 `0.3s ~ 0.5s`
  - `默认播放倍数`：默认 `1.0x`
  - `固定缩放倍数`：默认 `2`
- 仅在 `mark-v3` 详情页生效。
- 当脚本中心勾选 `隐藏平台AI功能` 时，自动隐藏两类平台原生 AI 板块：
  - 左侧 `AI 洞察` 卡片
  - 右下角平台 AI 浮动入口
- 当前运行时优先命中已采样的 `.insight-container-Hn0Gna` 与 `.trigger-wrapper-RlG7Dx`，未命中或命中过窄时，再按 `AI 洞察 / 统计周期 / 前往数据看板 / 立即生成` 文本锚点和右下角小型固定浮层特征兜底。
- 若详情工作区实际挂在同域 iframe 中，运行时也会继续下钻到 iframe 文档里做同样的显隐扫描。
- 页面发生重渲染后，会继续按当前开关补做显隐；若平台在同一节点上重写 `class/style`，也会重复压回隐藏状态。
- 页面首次加载和波形区重渲染后，会继续把播放速度控件和缩放输入框回填到当前脚本设置值。
- 当前会在波形区下方挂载 `苏州话脚本 Beta` 面板：
  - 读取当前条目的音频地址、模板和现有分段
  - 调用统一后端生成分段建议
  - 使用页面内捕获的 `SubmitTempItemAnswer` 暂存契约把新分段写回平台临时答案
  - 成功后自动刷新当前详情页，方便直接复核

## 运行时边界

- 当前只调用统一后端的分段预览接口：
  - `/api/bytedance-aidp/suzhou-helper/segment/preview`
- 当前不发识别类 AI 请求。
- 当前不新增快捷键、批量入口或自动提交。
- 当前不会直接点击平台 `提交 / 下一题 / 重置`。
- 当前写回只作用于 `SubmitTempItemAnswer` 暂存答案。
- 当前若发现分段表里已有转写文本或语音种类，或当前分段状态已变化，会 fail closed 停止自动应用。

## 文件

- `page-world/network-observer.js`
  - 捕获 `Receive` 当前条读取快照
  - 捕获 `SubmitTempItemAnswer` 暂存请求 URL、CSRF 头和基础请求体
- `data-api.js`
  - 解析详情页当前上下文
  - 校验当前分段是否已变化
  - 通过平台暂存接口应用分段建议
- `segmentation-controller.js`
  - 调用统一后端生成分段建议
  - 读取脚本配置里的前后静音补偿
- `ui-panel.js`
  - 渲染波形区下方的精简助手面板
- `content.js`
  - `mark-v3` 路由识别
  - 平台 AI 板块显隐
  - 波形区播放速度 / 固定缩放回填
  - 页面重渲染后的补隐藏
  - 分段建议面板协调与刷新

## 真实浏览器验证

1. 在仓库根目录启动统一后端：`node platform-resources/backend/server.js`
2. 在真实 Edge / Chrome 重新加载 unpacked extension，并刷新 `mark-v3` 详情页。
3. 进入脚本中心 `ByteDance AIDP -> 苏州话脚本`，确认：
   - `隐藏平台AI功能` 默认已勾选
   - `前后静音时长` 默认 `0.5`
   - `默认播放倍数` 默认 `1.0`
   - `固定缩放倍数` 默认 `2`
4. 回到详情页，确认左侧 `AI 洞察` 卡片和右下角平台 AI 浮动入口消失，同时波形区下方出现 `苏州话脚本 Beta` 面板。
5. 确认波形区的播放速度控件与缩放输入框会自动回填到设置值。
6. 点击 `生成分段建议`，确认面板出现建议段列表。
7. 点击 `应用分段建议`，确认页面刷新，分段表与波形区回显新的分段。
8. 再次复核左侧任务列表、波形区、`是否保留` 和分段表格未被隐藏或破坏。
