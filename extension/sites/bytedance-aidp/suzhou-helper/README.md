# ByteDance AIDP 苏州话脚本运行时

## 命中范围

- 站点：`https://aidp.bytedance.com/*`
- 目标页：`/management/task-v2/{taskId}/mark-v3/{index}`

## 当前能力

- 读取脚本中心里的基础设置：
  - `隐藏平台AI功能`：默认勾选并隐藏平台 AI 板块，取消勾选后才显示
  - `生成后自动应用当前建议`：默认开启；只在手动点击 `生成分段建议` 后，立即尝试写回当前暂存答案
  - `前后静音时长`：默认 `0.3s`，可调范围 `0s ~ 0.5s`
  - `静音阈值`：默认 `-31 dBFS`，只允许整数 `-80 ~ -5`
  - `连续相接自动合并`：默认开启；相邻建议段首尾相接或只差极小显示误差时自动并回一段
  - `默认播放倍数`：默认 `1.0x`，只允许选择平台系统预设 `0.5 / 0.75 / 1.0 / 1.25 / 1.5 / 1.75 / 2.0`
  - `固定缩放倍数`：默认 `2`，只允许选择整数 `1~10`
  - `快捷键`：默认全部未设置；按需录制后才生效
- 仅在 `mark-v3` 详情页生效。
- 当脚本中心勾选 `隐藏平台AI功能` 时，自动隐藏两类平台原生 AI 板块：
  - 左侧 `AI 洞察` 卡片
  - 右下角平台 AI 浮动入口
- 当前运行时优先命中已采样的 `.insight-container-Hn0Gna` 与 `.trigger-wrapper-RlG7Dx`，未命中或命中过窄时，再按 `AI 洞察 / 统计周期 / 前往数据看板 / 立即生成` 文本锚点和右下角小型固定浮层特征兜底。
- 若详情工作区实际挂在同域 iframe 中，运行时也会继续下钻到 iframe 文档里做同样的显隐扫描。
- 页面发生重渲染后，会继续按当前开关补做显隐；若平台在同一节点上重写 `class/style`，也会重复压回隐藏状态。
- 页面首次加载和波形区重渲染后，会继续处理播放速度控件和缩放输入框：
  - 播放速度只在进入当前详情页或切到新题时，按一次配置值走平台原生 `arco-select` 提交；成功后不再持续改回
  - 若点选目标倍速后平台展示值仍未变化，会补一次原生确认动作
  - 缩放只在进入当前详情页时按一次配置值；优先读取 `aria-valuenow`，再按平台原生缩小 / 放大按钮收敛到整数 `1~10`
- 当前会在波形区下方挂载 `苏州话脚本 Beta` 面板：
  - 读取当前条目的音频地址、模板和现有分段
  - 调用统一后端生成分段建议
  - 支持页内切换 `生成后立即应用当前建议`
  - 使用页面内捕获的 `SubmitTempItemAnswer` 暂存契约把新分段写回平台临时答案
  - 新写回的 region 默认带 `ms: "目标方言"`；仅在应用分段建议时自动带值，不再持续扫描页面空语言种类
  - 成功后自动刷新当前详情页，方便直接复核
  - `当前音频` 区默认折叠，并支持当前页内临时 `展开 / 折叠`
- 当前会在波形工具条 `.btns-play` 右侧挂两个辅助按钮：
  - `清空画段`
  - 先弹二次确认
  - 再通过 `SubmitTempItemAnswer` 直接把当前题 `regions` 清空
  - 不再额外判断当前是否已有转写文本或语音种类；也不直接提交或切题
  - `填充语言种类`
  - 只处理当前题当前页里空值或 `请选择` 的 `语言种类`
  - 通过已捕获的 `SubmitTempItemAnswer` 暂存契约直写当前题 `regions[*].ms = "目标方言"`
  - 只补空值，不覆盖已有非空语言；成功后会刷新当前详情页复核
- 当前支持 7 个详情页快捷键动作：
  - `播放/暂停切换`
  - `区间播放`
  - `回到首帧`
  - `删除当前选区`
  - `清空画段`
  - `生成分段建议`
  - `应用分段建议`
  - 所有快捷键都只点击页面真实按钮；按钮缺失或 disabled 时直接 fail closed

## 运行时边界

- 当前只调用统一后端的分段预览接口：
  - `/api/bytedance-aidp/suzhou-helper/segment/preview`
- 当前分段建议按“明显长静音优先作为边界，保留静音核心”的固定策略生成，不额外暴露 dbfs 可视调参控件
- 当前分段建议支持在脚本中心直接调 `静音阈值`，默认仍按 `-31 dBFS` 起步；若开启 `连续相接自动合并`，同一原始段里首尾实际相接的建议段会在预览阶段直接并回一段
- 当前不发识别类 AI 请求。
- 当前不做识别类批量入口或自动提交。
- 当前不会直接点击平台 `提交 / 下一题 / 重置`。
- 当前写回只作用于 `SubmitTempItemAnswer` 暂存答案。
- 当前若发现分段表里已有转写文本或语音种类，或当前分段状态已变化，会 fail closed 停止自动应用。

## 文件

- `page-world/network-observer.js`
  - 捕获 `Receive` 当前条读取快照
  - 捕获 `SubmitTempItemAnswer` 暂存请求 URL、CSRF 头和基础请求体
- `data-api.js`
  - 解析详情页当前上下文
  - 校验当前分段是否已变化
  - 通过平台暂存接口应用分段建议
- `segmentation-controller.js`
  - 调用统一后端生成分段建议
  - 读取脚本配置里的前后静音补偿、静音阈值与连续并段开关
  - 默认按 `-31 dBFS / 400ms / 0s~0.5s` 口径请求 AIDP 分段预览
- `ui-panel.js`
  - 渲染波形区下方的精简助手面板
  - 管理 `当前音频` 区的默认折叠状态
  - 管理 `生成后立即应用当前建议` 页内开关
- `shortcuts.js`
  - 绑定 AIDP 详情页快捷键
  - 输入框、下拉录入态和可编辑区域聚焦时自动忽略快捷键
- `content.js`
  - `mark-v3` 路由识别
  - 平台 AI 板块显隐
  - 波形区播放速度 / 固定缩放回填
  - 详情页快捷键与手动语言补齐按钮
  - 固定缩放的按钮链路同步
  - 页面重渲染后的补隐藏
  - 分段建议面板协调与刷新

## 真实浏览器验证

1. 在仓库根目录启动统一后端：`node platform-resources/backend/server.js`
2. 在真实 Edge / Chrome 重新加载 unpacked extension，并刷新 `mark-v3` 详情页。
3. 进入脚本中心 `ByteDance AIDP -> 苏州话脚本`，确认：
   - `隐藏平台AI功能` 默认已勾选
   - `生成后自动应用当前建议` 默认已勾选
   - `前后静音时长` 默认 `0.3`
   - `静音阈值` 默认 `-31`
   - `连续相接自动合并` 默认已勾选
   - `默认播放倍数` 为下拉框，默认 `1.00倍速`
   - `固定缩放倍数` 为下拉框，默认 `2`
   - 快捷键面板默认全部未设置
4. 回到详情页，确认左侧 `AI 洞察` 卡片和右下角平台 AI 浮动入口消失，同时波形区下方出现 `苏州话脚本 Beta` 面板。
5. 确认 `当前音频` 区默认折叠，且只保留一个 `展开当前音频 / 折叠当前音频` 按钮。
6. 确认波形区的播放速度控件会在进入详情页时按系统预设倍速做一次原生提交；若平台只完成点选但未真正生效，脚本会补一次确认动作。倍速成功后，脚本不会在同一题里持续改回；缩放会在进入详情页时通过平台原生缩放按钮实际收敛到设置值，而不只是输入框显示变化，后续手动改缩放时脚本也不会持续改回。
7. 点击工具条右侧 `清空画段`，确认先出现二次确认；确认后当前题分段被清空并刷新页面。
8. 点击工具条右侧 `填充语言种类`，确认脚本会通过平台暂存接口只补空值或 `请选择` 的行，已有非空语言保持不变，随后刷新页面复核。
9. 点击 `生成分段建议`，确认面板出现建议段列表。
10. 若开启 `生成后自动应用当前建议`，确认生成成功后会直接刷新页面；若关闭，再点击 `应用分段建议`，确认页面刷新，分段表与波形区回显新的分段。
11. 复核只有在“应用分段建议”写回后，新生成分段才会默认带 `目标方言`；手动画段不会再被脚本持续自动改语言。
12. 再次复核左侧任务列表、波形区、`是否保留` 和分段表格未被隐藏或破坏。
