# ByteDance AIDP 苏州话脚本运行时

## 命中范围

- 站点：`https://aidp.bytedance.com/*`
- 目标页：`/management/task-v2/{taskId}/mark-v3/{index}`

## 当前能力

- 读取脚本中心里的基础设置：
  - `隐藏平台AI功能`
  - `生成后自动应用当前建议`
  - `前后静音时长`
  - `静音阈值`
  - `连续相接自动合并`
  - `默认播放倍数`
  - `固定缩放倍数`
  - `启用普通话听写识别`
  - `AI 请求超时`
  - `听音 / 普通话听写收口` 两阶段模型、Prompt 与生成参数
  - `快捷键`
- 仅在 `mark-v3` 详情页生效。
- 当脚本中心勾选 `隐藏平台AI功能` 时，自动隐藏两类平台原生 AI 板块：
  - 左侧 `AI 洞察` 卡片
  - 右下角平台 AI 浮动入口
- 页面首次加载和波形区重渲染后，会继续处理播放速度控件和缩放输入框：
  - 播放速度只在进入当前详情页或切到新题时按一次配置值走平台原生提交
  - 若点选目标倍速后平台展示值仍未变化，会补一次原生确认动作
  - 缩放只在进入当前详情页时按一次配置值；优先读取 `aria-valuenow`，再按平台原生缩小 / 放大按钮收敛到整数 `1~10`
- 当前会在分段表格里补挂行内识别入口：
  - 表头新增一列 `识别音频`
  - 每行在现有 `暂停` 右侧新增一个 `识别音频` 按钮
  - 点击后只识别对应段，并直接写回该行 `regions[*].txt`
  - 成功后底部 `AI信息` 会展示本次 `最终普通话听写稿`、`听音阶段结果`、阶段级 token / 预估人民币，以及 `raw / debug`
- 当前会在波形区下方挂载 `苏州话脚本 Beta` 面板，固定为 4 个区块：
  - 第一排左侧：`分段建议`
    - 调用统一后端生成当前音频分段建议
    - 支持页内切换 `生成后立即应用当前建议`
    - 通过页面内捕获的 `SubmitTempItemAnswer` 暂存契约写回
  - 第一排右侧：`批量识别`
    - 只处理当前题当前页 `regions`
    - 默认全选，支持段号点选和拖选
    - 默认并发 `5`
    - 支持 `停止批量`
    - 整批识别结束后只合并成功且有非空结果的段，一次性调用一次 `SubmitTempItemAnswer`
  - 第二排整宽：`当前音频`
  - 第三排整宽：`AI信息`
    - 展示行内识别或批量识别最近一次结果的两阶段模型、usage、cost、raw、debug
- 当前会在波形工具条 `.btns-play` 右侧挂两个辅助按钮：
  - `清空画段`
  - `填充语言种类`
- 当前支持 7 个详情页快捷键动作：
  - `播放/暂停切换`
  - `区间播放`
  - `回到首帧`
  - `删除当前选区`
  - `清空画段`
  - `生成分段建议`
  - `应用分段建议`

## 写回契约

- AI 最终结果固定写入 `TempAnswer.Content.data.regions[*].txt`
- 最终结果字段固定为 `finalMandarinText`
- 不新增第二个结果字段
- 不改 `regions[*].ms`
- 不改提交 / 下一题链路
- 空结果处理：
  - 目标段原本已有非空 `txt` 时，空结果不覆盖，直接跳过
  - 目标段原本为空时，空结果视为 no-op

## 文本口径

- 最终输出是“普通话听写稿”，不是苏州话原文稿，也不是润色稿
- 普通话不要截取，听到多少写多少
- 以听音为主，不做语义扩写
- 标点只允许使用 `，。？！`，句末只允许使用 `。？！`
- 不知名人名、地名、公司名或其他无法精准锁定的事物，使用 `##名称##` 包起来
- 抖音音效不截取；主说话人如果在唱歌，也不截取这部分内容
- 语气词等按听到的普通话写法保留
- 明显口吃式连续重复最多保留 `3` 次
- 正常有语义的重复内容按听到的保留
- 不使用阿拉伯数字，统一改写为汉字数字
- 纯静音或完全听不清时返回空字符串

## 分段留白

- `前后静音时长` 的运行时补偿始终限制在 `0 ~ 500ms`
- 截取区间开头和结尾的无人声留白都不能超过 `500ms`

## 运行时边界

- 当前统一后端接口：
  - `/api/bytedance-aidp/suzhou-helper/segment/preview`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend/health`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend/defaults`
  - `/api/bytedance-aidp/suzhou-helper/ai/recommend`
- 当前批量识别只处理当前题当前页，不跨页、不自动切题
- 当前写回只作用于 `SubmitTempItemAnswer` 暂存答案
- 当前不会直接点击平台 `提交 / 下一题 / 重置`
- 当前不会新增平台提交 API
- 当前只改 `txt`，不改 `ms`
- 当前若发现当前题或当前快照已变化，批量写回会 fail closed 取消

## 文件

- `page-world/network-observer.js`
  - 捕获 `Receive` 当前条读取快照
  - 捕获 `SubmitTempItemAnswer` 暂存请求 URL、CSRF 头和基础请求体
- `data-api.js`
  - 解析详情页当前上下文
  - 通过平台暂存接口写回当前段 / 批量 `txt`
  - 应用分段建议
  - 清空画段
  - 填充语言种类
- `ai-recommendation.js`
  - 裁当前段音频
  - 调统一后端苏州话 AI 推荐接口
  - 复用共享 AI 使用人元数据
- `segmentation-controller.js`
  - 调统一后端生成分段建议
- `ui-panel.js`
  - 渲染分段建议 / 批量识别 / 当前音频 / AI 信息区块
- `shortcuts.js`
  - 绑定 AIDP 详情页快捷键
- `content.js`
  - `mark-v3` 路由识别
  - 平台 AI 板块显隐
  - 波形区播放速度 / 固定缩放回填
  - 分段表格行内识别按钮挂载
  - 面板协调、识别、批量识别、刷新与按钮接线

## 真实浏览器验证

1. 在仓库根目录启动统一后端：`node platform-resources/backend/server.js`
2. 在真实 Edge / Chrome 重新加载 unpacked extension，并刷新 `mark-v3` 详情页。
3. 进入脚本中心 `ByteDance AIDP -> 苏州话脚本`，确认基础设置和 AI 设置都能正确回显。
4. 回到详情页，确认左侧 `AI 洞察` 卡片和右下角平台 AI 浮动入口按开关显隐。
5. 确认 `苏州话脚本 Beta` 第一排左侧是 `分段建议`、右侧是 `批量识别`，下方依次是 `当前音频` 和 `AI信息`，且不再出现 `当前段识别` / `写回当前段`。
6. 在分段表格里确认表头新增 `识别音频`，并且每行 `暂停` 右侧都有 `识别音频` 按钮。
7. 点击任意一行 `识别音频`，确认只更新该行 `txt`，不改 `ms`，并在底部 `AI信息` 看到本次 `最终普通话听写稿` 和 `听音阶段结果`。
8. 验证纯静音或完全听不清时：
  - 原本已有 `txt` 的段不会被空结果覆盖
  - 原本为空的段保持不写
9. 验证 `批量识别并填入`：
  - 全选可用
  - 段号点选和拖选可用
  - `停止批量` 可用
  - 最终只发一次批量写回
10. 点击 `生成分段建议` 和 `应用分段建议`，确认分段建议链路仍可用。
11. 点击 `清空画段` 和 `填充语言种类`，确认旧能力不回归。
