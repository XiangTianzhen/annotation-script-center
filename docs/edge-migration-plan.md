# Edge 扩展迁移计划

## 当前项目目标

当前项目的目标不是复刻旧工程目录，而是把现有能力逐步迁移为一个可独立维护的 Edge 扩展项目。

当前项目的明确目标是：
- 以 `extension/` 为正式扩展代码根目录持续演进。
- 以 `docs/` 固化迁移规则、边界和阶段性结论。
- 以 `legacy-reference/` 保存旧油猴与旧服务端参考代码。

## 当前阶段判断

截至 2026-04-17，当前项目已经完成：
- 独立项目根目录收敛
- 核心文档基线建立
- 最小 MV3 扩展骨架建立
- `Alibaba LabelX` 的只读存在感注入建立
- 最小 options / popup / storage 骨架建立
- 共享存储层与非站点消费者收口
- 站点层页面契约统一
- 页面识别层落地
- SPA 路由观察层落地
- 只读页面状态采集层落地
- 只读 DOM 适配层首版落地
- 站点上下文提取稳定化落地
- `content.js` 装配层继续收口落地
- `task-detail` item 级只读快照落地
- `AnnotationValidationResult` 只读校验辅助落地
- `AnnotationFeedbackSummary` 只读反馈消费层落地
- 单条 target textarea 最小本地写入落地

截至当前，仍然明确未完成：
- 单条 validity 最小切换（不保存）
- 文本处理与 AI 标点修复迁移
- 保存链路迁移
- 自动化动作迁移

站在“先把当前单脚本迁进扩展”这个目标上，当前地基已经够用。
因此从现在开始，迁移计划应从“继续搭地基”切换为“单脚本迁移主线”。

### 2026-04-22 补充

当前 `Alibaba LabelX` 迁移已经进入高风险能力收口阶段，并完成首轮 MV3 等价落地：

- `document-start` 页面级 hook 已改为 `document-start.js -> page-world-hook.js -> isolated world bridge` 的 MV3 双世界桥接。
- 保存链不再只依赖页面按钮 click，已接入 pending saves、DOM 回读构 payload、手动安全保存、提交前强制安全保存。
- 智能提交、保存后刷新、列表页下一任务流转、自动批量提交闭环、自动抢单前端入口均已接入扩展运行时。
- legacy server 前端能力已接入扩展：AI 标点修复、词库同步/上传待审、排行榜、导出统计上传、必要的用户信息/时长上报。
- 旧油猴 `updateURL/downloadURL` 自更新已替换为扩展可用的“手动检查更新”方案：优先调用浏览器更新检查，远端版本清单只作为可选补充，不再在运行时下载覆盖脚本。

当前未完成项已经收敛为：

- 真实页面回归验证与少量 selector / 时序问题修补。
- 导出能力与旧脚本日期范围 / Excel 形态的完全等价仍待后续增量。
- 扩展发布链路需要单独提供更新包与版本清单，不再沿用旧 userscript 生命周期。

## 迁移总原则

- 当前项目根目录是唯一基准，不再假定存在旧外层工程。
- 每一轮先更新 `docs/`，再做代码改动。
- 旧代码只作为参考实现，不直接整段复制。
- 优先迁移低副作用、只读能力。
- 先建模块边界，再迁具体业务逻辑。
- 每一轮只交付一个可验证的小增量。

## 哪些地基现在可以停止继续扩写

以下内容当前不应再作为优先任务反复扩写：
- 为未来第二个平台预做更复杂抽象
- 为未来多脚本中心预做更复杂共享基础设施
- 在没有真实迁移压力前继续重构 `content.js`
- 提前处理排行榜、服务端对接和词库链路

原则：
- 只有当前单脚本迁移真正撞到边界时，才补最小必要地基。

## 当前最优先的下一步

当前最优先的是：
- 真实页面验证高风险链路

原因：
- 当前代码已进入保存链、提交链、云端能力和列表自动化的首轮整合状态。
- 这一阶段最大的风险不再是“是否能写一个最小入口”，而是“pending save、手动保存、提交前安全保存、自动流转是否在真实页面上保持可解释和可回读”。
- 因此下一步应优先做真实页面验证、日志复盘与小步修补，而不是继续扩写新抽象层。

## 当前是否需要拆成两个任务

结论：
- 当前更适合保持单任务。

原因：
- validity 切换的定位与切换语义共享同一套 radio 命中规则。
- 如果拆成“目标控件定位”和“切换语义”两个任务，会让接口在两个任务间反复调整。

## 当前明确不建议优先进入

- fetch 拦截
- 保存链路
- 自动提交、自动抢单、批处理
- 需要大量依赖旧逻辑整体搬运的模块

## 下一轮建议
