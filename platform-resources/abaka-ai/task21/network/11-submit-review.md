# Task21 送审按钮链路（快捷键 7）

## 结论

- 快捷键 `7` 只点击页面真实“送审 / Submit / Submit Review”按钮。
- 扩展不直接调用送审接口，不主动 `fetch`。
- 若出现二次确认弹窗，扩展不自动确认，需要用户手动确认。

## 触发方式

1. 用户在 Task21 `/items` 标注页手动按下快捷键 `7`。
2. 扩展在 DOM 中定位并点击页面真实送审按钮。
3. 后续 Network 请求由平台页面自身行为触发。

## 运行时限制

- `viewMode=true` 查看页不执行。
- 页面未检测到 `same_font` 字段时不执行。
- 疑似标注内审环境（如“标注内审 / 领取审核 / Claim Review / Reviewer / Review Team”）会阻止执行。

## 采集边界

- 记录结构，不记录敏感值。
- 不提交原始 HAR、完整响应体、客户原始数据。
