## 定位策略

后续脚本不能只依赖中文文案。建议优先级：

1. URL route 与 query keys，例如 `/task-v2/data-item`、`/items`、`taskId`、`itemId`、`nodeId`、`role`、`viewMode`。
2. 页面区域结构，例如左侧导航、面包屑、筛选区、表格区、右侧条目列表、same_font 主标注区。
3. 稳定属性，例如 `role`、`aria-label`、`name`、`data-col-key`、输入框 `placeholder`。
4. 中文 / English 双语文案兜底，按钮必须结合区域和动作风险判断。
5. 禁止只使用 `data-v-*`、hash class 或完整资源 URL 作为唯一定位依据。
