# 柳州话标注规则整理稿

来源：用户提供的 `柳州话标注规范_5.12.docx` 与项目说明，已整理为仓库内脱敏规则资产。

## 全局有效性

- 转写前先判断全局是否可标注。
- 若可标注，选择 `是（Valid）`。
- 若不可标注，需要先反馈问题，确认后再决定 `是（Valid）` 或 `否（Invalid）`。
- 不可未经确认直接选择 `否（Invalid）`。

## 画段有效性

- 语句可正常转写、语音清晰、无明显杂音或噪音时，可正常画段，并选择 `是（Valid）`。

## 画段无效性

- 原则上无效部分也要画段，并使用以下标签：
  - `<Silence>`
  - `<Unintelligible>`
  - `<Meaningless>`
- 只有静音段可以不画段、不打 `<Silence>`，但必须保证有效段落前后静音不超过 `0.2s`。

## 分段规则

- 连续静音阈值：`-27 dB`
- 静音 `< 0.4s`：不切
- 静音 `0.4s ~ 1.5s`：结合句意，能不切就不切；若不破坏整句语义，可切
- 静音 `> 1.5s`：必切
- 保留有效段前后静音：`0.1s ~ 0.2s`
- 单段时长原则：`0.1s ~ 15s`

## 转写原则

- 文本必须和声音完全一致，发什么写什么。
- 结巴、重复、口误等，柳州话文本按实际发音保留。
- 普通话顺滑句子需要删除结巴、口误等非必要内容，只保留顺滑表达。

## 数字与外来语

- 数字、型号、网址、邮箱等，按实际读音转写。
- 外来语按常用书写规则保留。
- 中文与外来语之间保留空格。

## 标点

- 本项目输出只允许：
  - `，`
  - `。`
  - `？`
  - `！`
- 不使用书名号 `《》`。

## 特殊标签

- 有效填充词标签：
  - `#um`
  - `#hmm`
  - `#ah`
  - `#eh`
- 当前标签联动范围只处理以上 4 个口语词标签，以及下方的 `<SPK/> / <NPS/>`。
- 当前自动推断只做高置信场景：
  - 独立 `呃 / 诶 / 欸` -> `#eh`
  - 独立 `啊` -> `#ah`
  - 独立 `嗯` -> `#um`
  - 重复笑声 `呵呵 / 哈哈 / 嘿嘿 / 嘻嘻` 一类主说话人非语义声音 -> `<SPK/>`
- `#hmm` 与 `<NPS/>` 当前不从纯文本自动猜测，只接受模型显式输出。
- 这些标签不可单独作为完整文本。
- 只出现语气、噪音的整段，应标为无效，并使用 `<Meaningless>`。

## 其他标签

- `<SPK/>`：主说话人的非语义噪音
- `<NPS/>`：非主说话人的非语义噪音
- `<Unintelligible>`：听不清、无法转写
- `<Meaningless>`：无意义声音或语气段

## 标签边界

- `<SPK/>`、`<NPS/>` 只应出现在有效句子中。
- 不能单独作为无效句子的全部内容。
- 当前页面联动里，标签只能写在 `标注文本`，不能写进 `普通话顺滑`。
- 标签前后不能同时各保留一个标点：
  - 正确：`你#ah，他弄死。`
  - 错误：`你，#ah，他弄死。`
- 若出现 `标点 + 标签 + 标点`，当前以标签后的标点为准，去掉标签前的标点。

## 普通话顺滑

- 需把柳州话转写再顺滑改写为普通话。
- 不加额外标签。
- 若柳州话里保留了 `#eh / #ah / #um / #hmm`，普通话顺滑需保留对应纯文本语气词。
- `<SPK/> / <NPS/>` 和笑声文本不保留到普通话顺滑里。
- 删除结巴、口误等非必要内容。
