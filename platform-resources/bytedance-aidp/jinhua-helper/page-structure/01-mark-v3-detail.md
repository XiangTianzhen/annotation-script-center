## 页面总览

- 当前已能从截图、HTML 片段和页面快照安全确认，这是一个“左侧任务列表 + 中间波形与字段表格 + 右下平台 AI 浮层”的详情标注页。
- 首轮按后续脚本接线需要拆成：
  - 顶部任务 / 条目上下文区
  - 左侧任务列表区
  - 波形与播放控制区
  - 脚本自有面板挂载区
  - `是否保留` 单选区
  - 分段表格区
  - 平台 AI 浮动触发器
  - 平台 AI 洞察面板
- 当前已可把“平台 AI”与“核心标注工作区”分层处理：基础设置 `隐藏平台AI功能` 只应影响平台 AI 板块，不应影响左侧任务卡、波形、保留/丢弃、分段表格。
- 当前已把“脚本自有面板”收口到波形区下方：用于展示当前音频摘要、分段建议按钮和建议回显，不直接侵入表格单元格。
- 当前脚本自有面板里的 `当前音频` 区默认折叠，且只在当前详情页内临时记忆 `展开 / 折叠` 状态。
- 当前已确认脚本还会在波形工具条 `.btns-play` 右侧追加一个 `清空画段` 按钮；该按钮属于脚本附加控制，不替换平台原生播放 / 撤销 / 删除按钮。

## 动态区域 / 重渲染风险

- 当前条目切换后，左侧任务卡、波形区、保留/丢弃和表格区可能联动重渲染
- 波形高亮区域、当前分段选中态、播放状态和表格行焦点都属于高频变化区
- 波形容器与分段表格切换时，脚本自有面板可能被平台重绘挤掉；运行时需要周期性补挂载
- 波形工具条里的倍速 `arco-select`、缩放输入框和 `.btns-play` 控制组都可能被平台整体重建；脚本对这三类节点都需要允许重试式补回填 / 补挂载
- 倍速 `arco-select` 当前不能只把“点到目标项”视为成功；平台可能还需要一次原生确认，运行时应以展示值真正变成目标倍速作为完成口径
- 缩放输入框显示值不等于平台实际已生效缩放；运行时需要优先读 `aria-valuenow`，并在进入详情页时通过原生缩放按钮逐步收敛一次
- AIDP 当前看不到 dbfs 面板；分段建议只能按波形里的可视静音区反推，明显长静音应优先保留中间空白，不应被前后补偿完全吃掉
- 当前脚本中心已额外提供 `静音阈值` dB 输入和“连续相接自动合并”开关；后者默认开启，只用于把同一原始段里首尾实际相接的建议段在预览阶段并回一段，不应把存在真实短静音的分段误并
- 平台 AI 浮动触发器和 `AI 洞察` 面板可能按平台状态懒加载、折叠或重新挂载
- 平台还可能在同一节点上直接重写 `class/style/hidden/aria-hidden`，导致旧的显隐状态被覆盖
- 当前未排除详情工作区被包进同域 iframe 的实现方式；若真实页面如此，运行时需要同时扫描顶层文档和 iframe 文档
- 在真实页面补采前，避免依赖：
  - `nth-child`
  - 临时高亮态 class
  - 仅凭按钮顺序判断动作语义

## 可挂载点建议

- 如后续要接入详情页辅助 UI，优先选择：
  - 波形区与 `是否保留` 单选区之间的安全空白区
  - `是否保留` 单选区与分段表格之间的安全空白区
  - 分段表格外层的独立 sibling 节点
- 当前已采用：
  - 优先挂在 `.neeko-wavesurfer-warper.neeko-wavesurfer` 后方
  - 回退到同时包含 `播放速度 / 总时长` 的波形容器后方
  - 工具条级按钮优先挂在 `.btns-play` 最右侧
  - `当前音频` 区在脚本面板内默认折叠，不额外侵占波形主操作条
- 不建议：
  - 直接覆盖左侧任务卡片
  - 直接覆盖波形画布和播放器控制区
  - 直接混进平台主动作按钮组
  - 与 `.trigger-wrapper-RlG7Dx` 或 `.insight-container-Hn0Gna` 共用挂载容器
- 后续如果实现基础设置 `隐藏平台AI功能`：
  - 优先策略：先命中已采样的 `.trigger-wrapper-RlG7Dx` 与 `.insight-container-Hn0Gna`
  - 兜底策略：按 `AI 洞察` 文本锚点和右下角小型固定浮层特征回退到真实外层
  - 不应误隐藏左侧任务列表、波形区、保留/丢弃区和分段表格

## 写操作边界 / 未确认项

- 当前已从截图确认顶部存在 `暂存 / 重置 / 继续下一题 / 提交` 一类动作入口，但当前脚本不直接点击这些动作。
- 当前已从截图确认分段表格至少存在：
  - 转写文本输入
  - 语音种类下拉
  - 音频段播放
  - 行级操作
- 当前已确认脚本分段写回不会直接操作 DOM 输入框，而是改写平台暂存答案里的 `regions`
- 当前 `清空画段` 同样不直接删 DOM 节点，而是通过平台暂存写回把当前题 `regions` 置空
- 当前分段建议的静音边界属于后端固定规则，不在详情页直接暴露调参控件
- 当前详情页本身仍不提供 dbfs 面板；静音阈值与连续并段只由脚本中心配置透传，不改页面原生表单结构
- 在真实页面补采前，以下都应视为 `write-action`：
  - 顶部暂存 / 提交 / 下一题
  - `是否保留` 单选切换
  - 表格里的转写文本、语音种类和行级操作
  - 任何由平台 AI 面板触发的生成、跳转或分析动作
- 当前未确认表格输入区是普通 textarea、受控输入框、富文本还是混合容器。
- 当前自动应用分段的 fail closed 规则：
  - 分段表已有文本或语音种类时停止
  - 当前页面分段状态和预览源分段不一致时停止
