# 提交任务客户端校验阻断

## 记录目的

本记录说明在当前未完成包中直接点击 `提交任务` 时，前端先做必填校验，未发出 `commit` 请求。

这不是一个网络请求，但对后续采集和扩展逻辑很重要：并非每次点击提交按钮都会进入 `/commit`。

## 触发操作

- 当前详情页只完成部分题卡。
- 点击顶部 `提交任务` 主按钮。

## 观察结果

- 未新增 `POST /api/v1/label/center/subTask/<REDACTED_SUBTASK_ID>/commit`。
- 未发生页面跳转。
- 页面出现错误提示：

```text
有必填题目未填写，完成后才可以切换到下一页或提交任务！
```

## 与已采集提交成功链路的区别

| 场景 | 是否发出 `/commit` | 后续行为 |
| --- | --- | --- |
| 当前包未完成，直接点击提交 | 否 | 前端 toast 阻断，停留详情页 |
| 已完成包点击提交 | 是 | 发出 `/commit`，成功后可能自动领取或返回首页 |

## 字段推断

- 前端会在发起提交请求前检查当前任务是否存在必填题未完成。
- 校验条件至少覆盖 `哪个ASR更优` 必填单选题。
- 校验失败不依赖后端响应，因此 Network 面板无法得到失败 response。

## Content Script 建议

- 不要把“用户点击提交按钮”直接等同于“任务提交请求已发生”。
- 如果需要判断真实提交，应监听 `/commit` endpoint，而不是只监听按钮点击。
- 若脚本提供辅助提交或跳转功能，应先检查每条题卡必填单选是否已完成。
- 看到该 toast 时，应提示用户仍需补全必填题，而不是重试网络请求。
